# Pathway — файловый менеджер для macOS в духе проводника Windows

**Дата:** 2026-07-20
**Статус:** одобрено (дизайн-сессия с пользователем)
**Визуальный референс:** интерактивный прототип в `context/` (React-мокап «Atlas File Manager» + скриншоты в `context/screenshots/`)

## Зачем

Личный проект. Finder не устраивает отсутствием привычного по Windows опыта. Главные боли, которые закрывает Pathway (по приоритету):

1. **Адресная строка с полным путём** — видимый и редактируемый путь, как в проводнике Windows.
2. **Дерево папок в сайдбаре** — полноценное дерево каталогов с раскрытием, а не плоский список избранного.
3. Вырезать/вставить файлы (⌘X/⌘V), переименование по F2 и остальной привычный опыт проводника.

Монетизация — не сейчас; сначала довести продукт до состояния, которым автор сам пользуется вместо Finder. Дистрибуция в будущем — нотаризованный DMG вне App Store.

## Технология

**Swift + SwiftUI с точечными AppKit-вставками.** Рассматривались чистый AppKit (многословнее, медленнее итерации) и Tauri с реюзом React-прототипа (ненативные Quick Look, иконки, drag&drop, корзина — постоянная борьба с системой). Выбран нативный стек: для файлового менеджера критична глубокая интеграция с macOS, а прототип служит дизайн-спецификацией.

- Минимальная версия macOS: **15 (Sequoia)**.
- **Без песочницы (sandbox выключен).** Для работы нужен Full Disk Access, который пользователь выдаёт в Настройках macOS.
- Язык интерфейса: русский.

## Объём MVP (v0.1)

Критерий успеха: автор начинает пользоваться Pathway вместо Finder для повседневной навигации и операций с файлами.

Входит:

- Одно окно: сайдбар (дерево + избранное) + адресная строка + тулбар + таблица файлов + статус-бар.
- Дерево каталогов с ленивой подгрузкой при раскрытии; секции «Избранное» (Desktop, Documents, Downloads, Pictures и свои папки) и «Этот Mac» (дерево от `/`).
- Адресная строка: хлебные крошки (клик по сегменту — переход) ⟷ редактируемое текстовое поле пути (⌘L; Enter — переход, Esc — отмена).
- Навигация: назад/вперёд/вверх (⌘[ / ⌘] / ⌘↑), история переходов, двойной клик.
- Таблица файлов: колонки Имя / Дата изменения / Размер / Тип; сортировка кликом по заголовку; множественное выделение (⌘, ⇧); папки первыми.
- Операции: копировать ⌘C, вырезать ⌘X, вставить ⌘V, удалить в Корзину ⌫, переименовать F2 (инлайн), новая папка ⌘⇧N, открыть приложением по умолчанию (Enter/двойной клик), показать в Finder.
- Контекстное меню: Открыть, Копировать/Вырезать/Вставить, Переименовать, Новая папка, В Корзину, Показать в Finder.
- Drag & drop: внутри таблицы в папки, на папки дерева, обмен с Finder.
- Живое обновление содержимого при изменениях на диске (FSEvents).
- Скрытые файлы: по умолчанию скрыты, переключатель ⌘⇧.
- Светлая/тёмная тема — системная.

Не входит (следующие итерации): вкладки, двухпанельный режим (split view), поиск, Quick Look, массовое переименование, метки, сетевые серверы, фильтры по типу, панель предпросмотра, виды «сетка/список», настройки плотности.

## Архитектура

Три слоя, зависимости направлены сверху вниз (UI → State → Core). Core не знает про UI и тестируется отдельно.

### Core (без UI)

| Модуль | Ответственность |
|---|---|
| `FileItem` | Модель элемента: URL, имя, размер, даты, тип (UTType), признак папки/симлинка, системная иконка |
| `DirectoryLoader` | Асинхронное чтение содержимого папки с метаданными батчем |
| `DirectoryWatcher` | Наблюдение за текущей папкой через FSEvents; событие → перезагрузка списка |
| `FileOperations` | Копирование, перемещение, удаление в Корзину (`NSWorkspace.recycle`), переименование, создание папки — через `FileManager`/`NSWorkspace` |
| `PasteboardService` | Копировать/вырезать через системный `NSPasteboard` (file URLs) — совместимость с Finder в обе стороны |

### State

- `PaneState` — текущий путь, история навигации, выделение, сортировка, набор «вырезанных» путей. Методы: `navigate(to:)`, `goBack()`, `goForward()`, `goUp()`.
- `AppState` — избранное, настройки (показ скрытых файлов), тема.

Однонаправленный поток данных: действие пользователя → метод State → асинхронная работа Core → обновление `@Published`-свойств → обновление UI. `DirectoryWatcher` при внешних изменениях перезагружает список с сохранением выделения.

### UI (SwiftUI + AppKit-вставки)

- `MainWindow` — компоновка зон.
- `SidebarView` — SwiftUI-список: избранное + дерево с ленивой подгрузкой.
- `AddressBarView` — крошки и режим редактирования пути.
- `FileListView` — **`NSTableView` через `NSViewRepresentable`**: производительность на тысячах строк, нативная сортировка, инлайн-редактор имени (F2), контекстное меню, drag & drop. «Вырезанные» строки полупрозрачны.
- `StatusBarView` — количество папок/файлов, выделение, размер выделенного.

Раскладка окна:

```
┌────────────────────────────────────────────────┐
│ Тулбар: ◀ ▶ ↑ ⟳ │ адресная строка │            │
├───────────┬────────────────────────────────────┤
│ Сайдбар   │ Таблица файлов                     │
│ Избранное │ Имя │ Дата изм. │ Размер │ Тип     │
│ Дерево    │ ...                                │
├───────────┴────────────────────────────────────┤
│ Статус-бар: N папок, M файлов · выделено K     │
└────────────────────────────────────────────────┘
```

## Поведение операций

- **Копирование/перемещение** — асинхронно, UI не блокируется. Конфликт имён → диалог «Заменить / Оставить оба / Пропустить». Для больших файлов — тонкая полоска прогресса в статус-баре (отдельного окна прогресса в MVP нет).
- **Вырезать/вставить** — «вырезать» помечает файлы (полупрозрачность) и кладёт URL-ы в системный буфер; физическое перемещение — в момент вставки. Esc снимает пометку. Исчезнувший до вставки файл пропускается.
- **Удаление** — только в Корзину; безвозвратного удаления в MVP нет.
- **Переименование** — F2, инлайн; при старте выделяется имя без расширения; проверка занятого имени и недопустимых символов до применения.
- **Новая папка** — «Новая папка» с автонумерацией, сразу режим переименования.

## Обработка ошибок

Принцип: никаких молчаливых провалов и никаких падений.

- **Нет прав доступа** — заглушка «Нет доступа» в области списка + кнопка, ведущая в раздел Full Disk Access Настроек macOS. Первый запуск: мягкая просьба выдать Full Disk Access.
- **Ошибка операции** (диск переполнен, файл занят) — тост с понятным текстом; частичный результат не откатывается, в тосте видно, сколько объектов обработано.
- **Папка исчезла во время просмотра** — автопереход к ближайшему существующему родителю + тост.
- **Симлинки** — видны со стрелкой, переход к цели; битые видны, но не открываются.
- **Большие папки (10 000+)** — асинхронная загрузка, UI не блокируется; `NSTableView` переваривает без самодельной виртуализации.

## Тестирование

1. **Юнит-тесты (Swift Testing)** на Core и State: `DirectoryLoader` (чтение, сортировка, скрытые файлы), `FileOperations` (копирование, конфликты имён, переименование — на временных папках), `PaneState` (история, выделение), логика вырезать/вставить. Файловые операции — ядро надёжности.
2. **Ручной чек-лист UI** после каждой фичи: открыть → выделить → вырезать → вставить в другую папку → F2 → в Корзину. XCUITest в MVP не используется.

## Дорожная карта после MVP (ориентир, вне объёма спеки)

Вкладки → Quick Look и поиск → двухпанельный режим → массовое переименование → метки → сетевые серверы. Приоритеты уточняются по опыту использования MVP.
