# Настройки подключения к серверу

Дата: 2026-07-20

## Задача

Сейчас подключение к серверу одноразовое: логин и пароль вводятся только когда
сервер отказал в гостевом входе, и нигде не сохраняются. При следующем запуске всё
заново. Смонтированный том в интерфейсе неотличим от локального диска, изменить
настройки подключения негде.

Нужно: сохранять учётные данные, показывать серверы в сайдбаре и редактировать их
настройки через контекстное меню.

## Хранение

Три места с разной природой данных:

| Что | Где | Живёт |
|---|---|---|
| Логин, пароль | Keychain (`kSecClassInternetPassword`) | всегда, до явного удаления |
| Адрес, имя, флаг «гость» | UserDefaults (`ServerBookmarks`) | всегда |
| Точка монтирования | Память (`MountedServers`) | до перезапуска |

Ключ учётных данных — канонический адрес (`smb://nas.local/Общие`). Класс
`kSecClassInternetPassword` выбран, чтобы записи были видны в системной «Связке
ключей» и пользователь мог удалить их вне приложения.

Разделение существенно: закладка есть всегда, учётные данные опциональны (гостевой
вход их не создаёт), факт подключения не переживает перезапуск.

### ServerCredentials

Новый тип в `PathwayCore`. Операции: `save(user:password:for:)`, `load(for:)`,
`delete(for:)`. Протокол `CredentialStoring` — чтобы тесты не ходили в настоящую
Связку ключей.

### MountedServers

Список подключённых томов. При запуске сверяется с `/Volumes`, дальше обновляется
при подключении и отключении. Отвечает на вопрос «этот адрес сейчас смонтирован и
если да, то где».

## Подключение

Порядок выбора учётных данных:

1. Есть сохранённые для адреса → подключаемся с ними.
2. Закладка помечена «гость» → гостевой вход.
3. Ничего не известно → пробуем гостя; отказ → экран авторизации.

Повторное подключение к известному серверу становится молчаливым. Экран логина
появляется только когда предъявить нечего.

После успешного входа с включённой галочкой «Запомнить» логин и пароль пишутся в
Связку ключей самим приложением. Параметр `saveToKeychain` из `ServerMounter`
удаляется: он передавался в NetFS, но это никогда не было проверено и, судя по
поведению, не работало.

### Устаревший пароль

Если сохранённый пароль перестал подходить, монтирование вернёт ошибку авторизации.
Тогда открывается экран логина с подставленным именем пользователя, пустым паролем и
сообщением «Не удалось войти с сохранённым паролем». Молчаливый переход в общую
ошибку недопустим — пользователь не поймёт, почему вчера работавший сервер сегодня
не открывается.

### Отключение

`NSWorkspace.unmountAndEjectDevice`. Если на томе открыты файлы, система откажет —
показываем текст ошибки, не проглатываем.

## Интерфейс

### Разделение «Мест» и «Сети»

Сетевые тома показываются только в «Сети». В «Местах» остаются локальные диски:
отбор идёт по `volumeIsLocal`, том неизвестного типа считается локальным — пропустить
свой диск хуже, чем показать сетевой дважды.

Из этого следует второе требование: «Сеть» обязана показывать и тома, смонтированные
мимо приложения — через Finder или до запуска Pathway. Иначе такой диск исчезает из
сайдбара совсем: в «Местах» его уже нет, а закладки на него никто не создавал.
`MountedServers.adoptExistingMounts` читает таблицу монтирования (`getmntinfo`,
файловые системы без флага `MNT_LOCAL`) и добавляет найденное в список.

Один ресурс может быть смонтирован дважды — `/Volumes/MAIN` и `/Volumes/MAIN-1`.
Адрес у них общий, поэтому точкой монтирования считается первая: остальные обычно
следы прошлых подключений.

### Секция «Сеть»

Строка на каждую закладку и на каждый подхваченный том, ниже — «Подключение к
серверу». У подключённого сервера иконка активного диска и точка-индикатор, у
остальных приглушённая. У подхваченного тома закладки нет, поэтому пункт «Удалить из
списка» ему недоступен.

Левый клик: подключён — переход на том; не подключён — подключение с прогрессом в
строке (по сети это секунды, прогресс обязателен).

### Контекстное меню

Подключённый: Открыть · Отключить · Изменить настройки · Забыть пароль · Удалить из
списка.

Отключённый: то же, но вместо «Открыть» и «Отключить» — «Подключиться».

«Забыть пароль» скрывается, если пароля в Связке нет: пункт, который ничего не
делает, хуже отсутствующего.

Удаление из списка запрашивает подтверждение и стирает пароль из Связки — закладки
нет, значит учётным данным незачем оставаться.

### Форма настроек

Третий шаг `ConnectServerView`: `.editing(ServerBookmark)`. Поля — адрес,
переключатель гость/пользователь, логин, пароль, «Запомнить».

Логин подставляется из Связки. Пароль не подставляется: показывается плейсхолдер
«сохранён», и Связка меняется только если пользователь что-то ввёл. Читать
сохранённый пароль в поле формы незачем — это лишний повод его засветить.

Если сервер подключён, под кнопкой «Сохранить» строка «Изменения применятся при
следующем подключении». Адрес при этом редактируется свободно: размонтирование тома
как побочный эффект кнопки «Сохранить» — операция со слишком серьёзными
последствиями, чтобы делать её неявно.

## Тестирование

`CredentialStoring` и `Mounting` подменяются фейками, поэтому логика подключения
тестируется без сети и без Связки ключей.

Что покрываем:

- выбор учётных данных по трём правилам приоритета;
- устаревший пароль ведёт на экран логина с подставленным именем;
- сохранение только при включённой галочке;
- удаление закладки стирает пароль;
- пустое поле пароля в форме редактирования не затирает сохранённый;
- «гость» не создаёт записи в Связке.

Реальную работу с Keychain проверяем отдельным тестом на временной записи, помеченным
как ручной, — по образцу `SMBBenchmark`, который пропускается без смонтированного
тома.

## Границы

Не делаем: правый клик по тому в дереве «Места» (возможное продолжение),
автоподключение при запуске, множественные учётные записи на один адрес.
