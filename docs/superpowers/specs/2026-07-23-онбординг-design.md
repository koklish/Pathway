# Онбординг: пошаговое обучение при первом запуске

## Цель

Показать новому пользователю ключевые возможности «Проводника» сразу после
первого локального запуска — пошаговый тур с подсветкой реальных элементов
интерфейса. Тур можно перезапустить вручную кнопкой «?» в тулбаре.

Шесть шагов: приветствие → адресная строка → сайдбар → подключение к серверу →
обновления → готово. Три средних шага **подсвечивают** конкретный элемент окна
(затемнение фона + вырез с рамкой вокруг цели), крайние показывают карточку без
подсветки.

Тур **чисто обучающий**: overlay перехватывает все клики, взаимодействовать с
подсвеченным элементом нельзя. Навигация — только «Далее» / «Назад» /
«Пропустить», плюс Esc, клик по тёмному фону и стрелки ←/→.

Показ **только при первом запуске** (не при обновлении) — флаг в UserDefaults,
переживающий обновления. Обновление не сбрасывает UserDefaults, поэтому
повторно тур сам не появится.

## Архитектура

Граница PathwayCore / Pathway соблюдается: вся логика тура — в Core и
тестируется без UI; рисование и сбор координат — в Pathway. Модель следует
паттерну обратного канала «Core просит UI» (`AppState.pendingEditPath` как
образец) и паттерну флага первого запуска (`FavoritesStore` с ключом
`favorites.seeded`, `FavoritesStore.swift:31-44`).

### Модель шагов — `Sources/PathwayCore/Onboarding.swift` (новый файл)

`OnboardingTarget` — что подсвечивать. UI сопоставляет кейс координатам:

```swift
public enum OnboardingTarget: Equatable, Sendable {
    case none          // центр окна, без выреза (приветствие, готово)
    case addressBar    // адресная строка целиком
    case sidebar       // сайдбар / дерево папок целиком
    case connectServer // кнопка «Подключиться к серверу…»
    case versionChip   // чип версии в тулбаре — вырез невозможен, карточка в углу
}
```

`OnboardingStep` — шаг тура. Порядок = `rawValue`, прогресс «N из 6» из него же:

```swift
public enum OnboardingStep: Int, CaseIterable, Sendable {
    case welcome, addressBar, sidebar, connectServer, updates, done

    public var title: String { … }
    public var body: String { … }
    public var target: OnboardingTarget { … }
}
```

`.versionChip` — особый: чип живёт в тулбаре macOS, отдельном слое, куда
SwiftUI-overlay не дотягивается. Вырез вокруг него сделать нельзя, поэтому
карточка просто встаёт в правом верхнем углу под чипом, без выреза.

Тексты шагов (русский, под реальные функции «Проводника», не под макет
Pathfinder со скринов):

- **welcome** — «Добро пожаловать в Проводник». Файловый менеджер для macOS в
  духе Проводника Windows: редактируемый путь, дерево папок, вкладки. Быстро
  пройдёмся по главному.
- **addressBar** — «Адресная строка». Каждый сегмент пути кликабелен —
  мгновенный переход на любой уровень. Кликните по строке (или ⌘L), чтобы
  ввести путь текстом, как в проводнике.
- **sidebar** — «Дерево папок и избранное». Слева — «Этот Mac», диски и ваши
  папки. Часто нужные папки перетащите в «Избранное».
- **connectServer** — «Сетевые серверы». Подключайте удалённые диски по SMB,
  FTP, SFTP и WebDAV (⌘K). Серверы появляются в сайдбаре в разделе «Сеть».
- **updates** — «Обновления». Приложение само проверяет обновления. Точка рядом
  с номером версии — доступна новая версия; нажмите на неё, чтобы установить.
- **done** — «Готово». Это всё главное. Тур можно повторить кнопкой «?» справа
  вверху в любой момент.

### Управление туром — `OnboardingModel` (тот же файл)

```swift
@Observable
@MainActor
public final class OnboardingModel {
    public private(set) var currentStep: OnboardingStep?   // nil — тур не идёт
    private let defaults: UserDefaults
    private let shownKey = "onboarding.shown"

    public init(defaults: UserDefaults = .standard)

    public var isActive: Bool { currentStep != nil }

    /// Первый локальный запуск: если тур ещё не показывали — запустить и
    /// пометить показанным. Иначе ничего.
    public func startIfFirstLaunch()

    /// Ручной запуск с кнопки «?» — всегда с первого шага, флаг не трогает.
    public func start()

    public func next()   // done → finish()
    public func back()   // welcome → без изменений
    public func skip()   // = finish()
    private func finish() // currentStep = nil
}
```

`shownKey` устроен как `seededKey` в `FavoritesStore`: `startIfFirstLaunch`
читает флаг; если не установлен — `currentStep = .welcome` и
`defaults.set(true, forKey: shownKey)`. `start()` флаг не читает и не пишет —
повторный тур не зависит от первого запуска.

### Интеграция в AppState — `Sources/PathwayCore/AppState.swift`

Добавить свойство рядом с `browser`, `favorites`, `folderActions`:

```swift
public let onboarding: OnboardingModel
```

Создаётся в `AppState.init` (`AppState.swift:36-44`), параметром с дефолтом
`onboarding: OnboardingModel = OnboardingModel()` — по образцу `favorites`,
чтобы тесты подсовывали свой `UserDefaults`.

### Overlay подсветки — `Sources/Pathway/OnboardingOverlay.swift` (новый файл)

**Сбор координат целей.** `PreferenceKey`, собирающий якоря целей окна:

```swift
struct OnboardingAnchorKey: PreferenceKey {
    static let defaultValue: [OnboardingTarget: Anchor<CGRect>] = [:]
    static func reduce(...) { value.merge(...) { a, _ in a } }
}

extension View {
    func onboardingTarget(_ target: OnboardingTarget) -> some View {
        anchorPreference(key: OnboardingAnchorKey.self, value: .bounds) {
            [target: $0]
        }
    }
}
```

Метки вешаются на реальные вью:
- `AddressBarView(...)` в `MainWindow.swift:45` → `.onboardingTarget(.addressBar)`
- `SidebarView(...)` в `MainWindow.swift:29` → `.onboardingTarget(.sidebar)`
- Кнопку «Подключиться к серверу…» в `NetworkSection.swift:60-76` →
  `.onboardingTarget(.connectServer)`

**Слой подсветки.** `MainWindow` через `.overlayPreferenceValue(OnboardingAnchorKey.self)`
получает `[OnboardingTarget: Anchor<CGRect>]` и `GeometryProxy`, рисует
`OnboardingOverlay`, если `onboarding.isActive`:

- Затемнение `Color.black.opacity(0.5)` на весь `MainWindow`.
- Для целей `.addressBar` / `.sidebar` / `.connectServer` — вырез: затемнение
  собирается в `.compositingGroup()`, поверх рисуется `RoundedRectangle` по rect
  цели с `.blendMode(.destinationOut)` (дыра), затем accent-рамка/свечение по
  тому же rect. Для `.none` и `.versionChip` выреза нет.
- **Карточка** — заголовок «N из 6» (accent), title (bold), body, ряд снизу:
  «Пропустить» слева, точки-индикаторы по центру, «Назад» + «Далее»/«Готово»
  справа. Стиль по скринам: белая карточка, скругление, тень.
- **Позиция карточки:** рядом с rect цели, сторона (снизу/сверху/сбоку) по
  наличию места; `.none` — центр окна; `.versionChip` — правый верхний угол под
  тулбаром.

**Перехват и клавиши.** Затемняющий слой ловит все клики (`.contentShape` +
`.onTapGesture { onboarding.skip() }` вне карточки). Esc, ←/→ — через
`.onKeyPress` на overlay (или скрытые `Button`-ы с `.keyboardShortcut`), поднимает
фокус overlay при появлении. Смена шага и появление/исчезновение — с
`withAnimation`/`.animation`.

### Кнопка «?» и запуск — `Sources/Pathway/MainWindow.swift`, `PathwayApp.swift`

**Кнопка «?»** — отдельный `ToolbarItem(placement: .primaryAction)`, объявленный
**перед** блоком с `UpdateBadgeView` (`MainWindow.swift:73-82`), чтобы встать
слева от чипа. Иконка `questionmark.circle`, действие → `appState.onboarding.start()`.
Под `#available(macOS 26, *)` — с `.sharedBackgroundVisibility(.hidden)`, как у
бейджа (тот же комментарий про Liquid Glass, `MainWindow.swift:68-72`).

**Первый запуск** — `appState.onboarding.startIfFirstLaunch()` в
`PathwayAppDelegate.applicationDidFinishLaunching` (`PathwayApp.swift:49-54`):
срабатывает ровно один раз за процесс, в отличие от `MainWindow.onAppear`,
который повторяется при закрытии/открытии окна по ⌘W.

## Поток данных

1. Старт процесса → `applicationDidFinishLaunching` → `onboarding.startIfFirstLaunch()`.
   Если флаг `onboarding.shown` не стоит: `currentStep = .welcome`, флаг ставится.
2. `MainWindow.body` наблюдает `onboarding.currentStep`. `isActive` → рисуется
   `OnboardingOverlay` поверх окна.
3. Overlay берёт координаты цели текущего шага из `OnboardingAnchorKey`, рисует
   затемнение с вырезом и карточку.
4. «Далее»/«Назад»/стрелки → `onboarding.next()/back()` меняет `currentStep` →
   overlay перерисовывается на новую цель.
5. «Пропустить»/Esc/клик по фону/«Готово» на последнем шаге → `finish()` →
   `currentStep = nil` → overlay исчезает.
6. Кнопка «?» в тулбаре → `onboarding.start()` → `currentStep = .welcome`,
   тур идёт заново независимо от флага.

## Тесты — `Sources/PathwayCoreTests/OnboardingTests.swift` (новый)

swift-testing, `@Suite("Онбординг")`, изоляция UserDefaults через свежий
`suiteName` на UUID (как в проекте).

- `startIfFirstLaunch` при чистом UserDefaults запускает тур с `.welcome` **и**
  ставит флаг `onboarding.shown`.
- `startIfFirstLaunch` при установленном флаге тур **не** запускает
  (`currentStep == nil`) — обновление не показывает тур повторно.
- Два экземпляра `OnboardingModel` на одном UserDefaults: второй
  `startIfFirstLaunch` не запускает тур (флаг общий).
- `start()` запускает тур с `.welcome` даже при установленном флаге, флаг **не**
  трогает.
- `next()` идёт по шагам по порядку `welcome → … → done`.
- `next()` на `.done` завершает тур (`currentStep == nil`).
- `back()` возвращается на шаг назад; на `.welcome` остаётся `.welcome`.
- `skip()` завершает тур с любого шага.
- Каждый `OnboardingStep` имеет непустые `title` и `body`; `target` соответствует
  ожидаемому (welcome/done → `.none`, addressBar → `.addressBar`, и т.д.).

## Ручной чек-лист

- [ ] Чистый профиль (удалить ключ `onboarding.shown` из UserDefaults) → первый
      запуск показывает тур с приветствия.
- [ ] Повторный запуск (флаг стоит) — тур сам не появляется.
- [ ] Кнопка «?» в тулбаре слева от чипа версии запускает тур заново.
- [ ] Шаг 2: адресная строка обведена рамкой, остальное окно затемнено.
- [ ] Шаг 3: сайдбар подсвечен.
- [ ] Шаг 4: кнопка «Подключиться к серверу…» подсвечена (при необходимости
      секция «Сеть» видна).
- [ ] Шаг 5: карточка в правом верхнем углу указывает на чип версии, без выреза.
- [ ] «Далее»/«Назад», стрелки ←/→ листают шаги; точки-индикаторы совпадают с
      «N из 6».
- [ ] «Пропустить», Esc и клик по тёмному фону закрывают тур.
- [ ] По подсвеченному элементу нельзя кликнуть сквозь overlay (тур обучающий).
- [ ] Кнопка «?» не ломает раскладку тулбара и Liquid Glass под macOS 26.

## Вне рамок

- **Интерактивность подсвеченных элементов.** Тур чисто обучающий — overlay
  перехватывает клики. Причина: сквозной клик потребовал бы синхронизировать
  состояние окна с шагом тура и легко ломает навигацию.
- **Повторный показ при обновлении / по версии.** Только первый запуск + ручной
  запуск. Не просили; хранение по версии сложнее и назойливее.
- **Вырез вокруг чипа версии.** Чип в отдельном слое тулбара, SwiftUI-overlay до
  него не дотягивается — вместо выреза карточка в углу. NSPopover от чипа
  (как `BadgeChipHost`) отверг: другой визуальный стиль (стрелка-носик), не
  совпал бы с карточками остальных шагов.
- **Анимированные указатели-стрелки к элементам.** Достаточно рамки-выреза и
  карточки рядом.
- **Подсветка отдельных под-элементов** (конкретный сегмент пути, конкретная
  папка). Подсвечиваем вью целиком.
