# Локализованные имена системных папок

## Цель

Показывать стандартные папки macOS по-русски — «Рабочий стол» вместо `Desktop`, «Документы» вместо `Documents`, — как это делает Finder. Сейчас список файлов показывает сырое имя с диска, а сайдбар, вкладки и адресная строка локализуют имена сами по себе, каждый своим вызовом.

Задача двойная: добавить перевод в список файлов и свести три существующих способа к одному, чтобы имя одной и той же папки нигде не расходилось.

## Архитектура

### SystemFolderNames — словарь вместо опроса системы

Новый файл `Sources/PathwayCore/SystemFolderNames.swift`. Чистый тип без состояния:

```swift
public enum SystemFolderNames {
    public static func localizedName(for url: URL) -> String?
    public static func displayName(for url: URL) -> String
}
```

`localizedName` отдаёт русское имя только для точного совпадения полного пути, иначе nil. `displayName` — обёртка с откатом на `lastPathComponent`, её и зовёт весь UI.

**Ключ — полный путь, а не имя папки.** Иначе любая пользовательская папка `Documents` внутри проекта превратилась бы в «Документы», а это уже не перевод, а подмена чужого имени.

**Словарь, а не `FileManager.displayName`.** Замер на этой машине: `displayName` стоит 0.3–1.2 мс на вызов (88 мс на 75 элементов домашней папки, 15 мс на 45 элементов `/Applications`). На каталоге в 500 файлов это 150–600 мс поверх нынешних 0.7 мс быстрого прохода, а на сетевом томе — кратно больше. Правило «быстрый проход не трогает диск ни разу» (см. CLAUDE.md, «Загрузка каталога») этого не переживёт. Словарь стоит ноль обращений к диску.

Состав — домашние папки (`Desktop` → «Рабочий стол», `Documents` → «Документы», `Downloads` → «Загрузки», `Pictures` → «Изображения», `Music` → «Музыка», `Movies` → «Фильмы», `Public` → «Общие», `Library` → «Библиотеки», `Applications` → «Программы») и системные (`/Applications` → «Программы», `/Applications/Utilities` → «Утилиты», `/Library` → «Библиотеки», `/System` → «Система», `/System/Applications`, `/System/Library`, `/Users` → «Пользователи»). Домашний каталог берётся из `FileManager.homeDirectoryForCurrentUser` при первом обращении.

Пути перед сравнением приводятся к каноническому виду: снимается завершающий слэш, разрешается `/var` → `/private/var`. Регистр **не** игнорируется: APFS регистронезависима по умолчанию, но `Documents` и `documents` — разные записи на регистрозависимом томе, и переводить второе неверно.

### Точки применения

Четыре места, из них три уже существуют и переключаются на общую функцию:

- **`DirectoryLoader`** (новое) — оба места создания `FileItem`: быстрый проход (строка 59) и `item(for:)` (строка 101). Отсюда русские имена в списке файлов.
- **`BrowserModel.displayName`** (строка 103) — приватный хелпер хлебных крошек, тело заменяется вызовом `SystemFolderNames`.
- **`TabsModel.title`** (строка 29) — заголовок вкладки.
- **`Favorite.defaultName`** (строка 18) — имя при закреплении папки в избранном.

`SidebarModel` не трогается: его элементы либо именованы вручную («Этот Mac», «iCloud Drive»), либо это тома с `volumeName`. Избранное в сайдбаре рисуется из `Favorite`, то есть перевод приходит туда само.

### Что остаётся сырым

`FileItem.url` не меняется. На нём держатся файловые операции, сравнение выделения и инлайн-переименование — редактор берёт исходное имя из `renaming.lastPathComponent` (`FileListView.swift:354`, `426`), то есть из URL, а не из `name`. Поэтому переименование «Рабочего стола» покажет в редакторе `Desktop`, и записан на диск будет тот же `Desktop`.

Русское имя живёт строго в поле `name`, то есть только в отображении.

## Поток данных

```
DirectoryLoader.loadNames
  → SystemFolderNames.displayName(for: url)   ← словарь, без диска
  → FileItem(name: «Рабочий стол», url: …/Desktop)
  → BrowserModel.items
  → FileListView (показ) и BrowserModel.sortedItems (сортировка)
```

Сортировка идёт по `name` (`BrowserModel.swift:434`), поэтому «Рабочий стол» встаёт по букве «Р», а не по «D». Это совпадает с Finder, но выглядит как изменение порядка — названо здесь явно, чтобы не выяснять постфактум.

## Тесты

Новый файл `Tests/PathwayCoreTests/SystemFolderNamesTests.swift`:

- `~/Desktop` даёт «Рабочий стол», `~/Documents` — «Документы»
- `/Applications` даёт «Программы», `/Applications/Utilities` — «Утилиты»
- папка с системным именем в другом месте (`/tmp/Desktop`, `~/Projects/Documents`) перевода **не** получает — ключом служит полный путь, а не имя
- несистемная папка отдаёт nil из `localizedName` и `lastPathComponent` из `displayName`
- завершающий слэш не мешает совпадению
- регистр учитывается: `~/desktop` перевода не получает
- сам домашний каталог перевода не получает: `localizedName` отдаёт для него nil

В `DirectoryLoaderTests`:

- обычный временный каталог с файлом `Desktop` внутри отдаёт имя `Desktop` без перевода
- имя в `FileItem` совпадает с `lastPathComponent` для несистемных путей

## Ручной чек-лист

- [ ] В домашней папке видны «Рабочий стол», «Документы», «Загрузки», «Изображения», «Музыка», «Фильмы»
- [ ] Сортировка по имени ставит их по русским буквам
- [ ] Двойной клик по «Рабочему столу» открывает `Desktop`, адресная строка показывает «Рабочий стол»
- [ ] Переименование «Документов» (F2) показывает в редакторе `Documents`; Escape ничего не меняет
- [ ] Заголовок вкладки и пункт избранного показывают то же имя, что и список
- [ ] В `/Applications` видны «Утилиты»; в произвольном проекте папка `Documents` осталась английской
- [ ] Копирование и перемещение «Рабочего стола» работают — операции идут по URL

## Вне рамок

- **`FileManager.displayName` для всех папок подряд** — отвергнуто по замеру: 150–600 мс на каталог из 500 элементов против нынешних 0.7 мс, на SMB кратно хуже.
- **Проверка маркера `.localized` в каждой папке** — дешевле `displayName` (0 мс на 75 элементов), но это всё же `lstat` на каждый элемент, то есть сетевой запрос на SMB на каждый объект. Ради конечного и заранее известного списка папок платить обходом диска незачем.
- **Перевод папок сторонних приложений** с собственным `.localized` — следствие предыдущего пункта: без опроса диска их не найти.
- **Локализация имени домашней папки пользователя** — Finder показывает там имя учётной записи, а не перевод.
- **Ввод русского имени в адресной строке** — `PathInput` разбирает настоящие пути; принимать «Рабочий стол» как путь значило бы заводить обратный словарь и разрешать неоднозначности.
