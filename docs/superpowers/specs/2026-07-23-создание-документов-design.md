# Создание документов из контекстного меню

## Цель

Дать подменю «Создать» в контекстном меню списка файлов — как в Проводнике Windows. Кроме папки, оно создаёт готовые пустые документы: текстовый файл, RTF, документы Microsoft Office и iWork.

Список пунктов подстраивается под машину: пункт показывается, если приложение, которое открывает такой файл, действительно установлено. Пункты сгруппированы — базовые, Microsoft, Apple.

Документ создаётся мгновенно копированием заготовки из бандла: приложение-редактор при этом не запускается.

## Архитектура

### `Sources/PathwayCore/DocumentTemplates.swift` — что можно создать

Описание шаблона, без ввода-вывода:

```swift
public enum TemplateGroup: Sendable { case basic, microsoft, apple }

public struct DocumentTemplate: Identifiable, Sendable {
    public let id: String            // "docx" — он же имя файла-заготовки в Resources/Templates
    public let title: String         // "Документ Word"
    public let defaultName: String   // "Новый документ"
    public let fileExtension: String // "docx"
    public let group: TemplateGroup
    public let requiredApp: String?  // bundle ID; nil — пункт доступен всегда
}
```

Статический `all` перечисляет восемь шаблонов в порядке отображения:

| id | Заголовок | Имя по умолчанию | Группа | requiredApp |
|---|---|---|---|---|
| `txt` | Текстовый документ | Новый документ | `.basic` | нет |
| `rtf` | Документ RTF | Новый документ | `.basic` | нет |
| `docx` | Документ Word | Новый документ | `.microsoft` | `com.microsoft.Word` |
| `xlsx` | Книга Excel | Новая книга | `.microsoft` | `com.microsoft.Excel` |
| `pptx` | Презентация PowerPoint | Новая презентация | `.microsoft` | `com.microsoft.Powerpoint` |
| `pages` | Документ Pages | Новый документ | `.apple` | `com.apple.iWork.Pages` |
| `numbers` | Таблица Numbers | Новая таблица | `.apple` | `com.apple.iWork.Numbers` |
| `key` | Презентация Keynote | Новая презентация | `.apple` | `com.apple.iWork.Keynote` |

`.txt` и `.rtf` без `requiredApp`: TextEdit есть в любой macOS, проверять нечего.

Microsoft выше Apple намеренно: приложение раздаётся коллегам на рабочих машинах, где Office и есть рабочий инструмент.

Фильтрация — чистая функция:

```swift
public static func available(with apps: AppLookup) -> [DocumentTemplate]
```

Возвращает шаблоны, у которых `requiredApp == nil` или приложение установлено, сохраняя порядок `all`.

### `Sources/PathwayCore/InstalledApps.swift` — граница с Launch Services

По правилу проекта каждая граница с ОС прячется за протоколом:

```swift
public protocol AppLookup: Sendable {
    func isInstalled(bundleID: String) -> Bool
}
```

`InstalledApps` — боевая реализация. **Опрос происходит один раз, в инициализаторе:** он проходит по всем непустым `requiredApp` из `DocumentTemplates.all`, зовёт `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)` и складывает найденные в `Set<String>`. `isInstalled` дальше читает только этот набор.

Это осознанный компромисс. `menuNeedsUpdate` выполняется на каждый правый клик — обращаться оттуда к Launch Services шесть раз значит класть работу на горячий путь. Плата: Office, установленный при запущенном Проводнике, появится в меню лишь после перезапуска. Office ставят один раз, а меню открывают сотни раз за сеанс.

Отвергнута инвалидация по `NSWorkspace.didLaunchApplicationNotification` — сложность ради события, которого почти не бывает.

`StubAppLookup` в тестах отдаёт заданный набор и не трогает систему.

### `Resources/Templates/` — заготовки

Восемь пустых файлов, имя = `id` шаблона: `txt`, `rtf`, `docx`, `xlsx`, `pptx`, `pages`, `numbers`, `key`.

`.txt` — нулевой длины, `.rtf` — минимальный заголовок RTF. Остальные шесть — ZIP-контейнеры, и получены они по-разному:

- **`.docx`/`.xlsx`/`.pptx` собраны кодом** — это OOXML, то есть ZIP с несколькими XML внутри, и минимальный валидный документ пишется руками. Проверено запуском: Word открывает документ с одним абзацем, Excel — книгу с одним листом, PowerPoint — колоду с одним слайдом, без предупреждений о повреждении.
- **`.pages`/`.numbers`/`.key` сохранены приложениями** — формат закрытый, внутри бинарный IWA-снапшот, а не XML; сочинить его нельзя. Пустые документы сохранены один раз вручную и лежат в репозитории.

Заготовки — обычные файлы в репозитории: генератор OOXML в приложение не входит, при создании документа выполняется только копирование.

В `Package.swift` таргет `PathwayCore` получает `resources: [.copy("Resources/Templates")]`; читаются через `Bundle.module`.

Подмена форматов iWork вордовскими (пункт «Документ Pages» создаёт `.docx`) отвергнута по результату проверки: Pages показывает на таком файле диалог импорта, то есть создание документа перестаёт быть бесшовным. Ради этого и держим три отдельные заготовки.

Создание документа через AppleScript отвергнуто: запуск Office занимает секунды, требует разрешения на автоматизацию и ломается при обновлении Office — проект уже избегает Apple Events там, где есть альтернатива (см. `TerminalLauncher`). Приложения запускались один раз при разработке, чтобы получить и проверить заготовки; у пользователя не запускается ничего.

### `FileOperations.createDocument` — копирование заготовки

```swift
public func createDocument(_ template: DocumentTemplate, in directory: URL, templatesRoot: URL) throws -> URL
```

Берёт `templatesRoot/<id>`, копирует в `uniqueURL(in:name:)` с именем `<defaultName>.<fileExtension>`, возвращает URL созданного файла. Конфликт имён разрешается тем же `uniqueURL`, что и в `createFolder`, — «Новый документ 2.docx».

`templatesRoot` передаётся параметром, а не берётся из `Bundle.module` внутри: так операция тестируется на временной папке, без бандла.

Отсутствие заготовки — `FileOperationError`: это ошибка сборки, а не ситуация выполнения, но падать приложению незачем.

### `BrowserModel.createDocument` — обвязка

По образцу `createFolder()`, через тот же `run { }`, который ловит ошибку в `errorMessage` и перезагружает каталог. Отличие одно: URL созданного файла нужно вернуть наружу, чтобы UI открыл инлайн-редактор.

### `Sources/Pathway/FileListView.swift` — пункт меню

В `menuNeedsUpdate` на месте нынешнего `add(to: menu, .newFolder, …)` появляется подменю «Создать»:

```
Папка                        ⇧⌘N
─────────────────────────────
Текстовый документ
Документ RTF
─────────────────────────────
Документ Word
Книга Excel
Презентация PowerPoint
─────────────────────────────
Документ Pages
Таблица Numbers
Презентация Keynote
```

Разделители ставятся между группами, а не после каждой: если Office не установлен, лишней черты быть не должно. Правило — разделитель перед группой, если она непуста и до неё уже что-то выведено.

Пункты подменю неактивны на томе только для чтения — по тому же условию, что `.newFolder` в реестре.

«Папка» остаётся отдельной командой реестра со своим `⇧⌘N`: шорткат уже занят и меняться не должен. Прочие пункты в `CommandRegistry` не заводятся — реестр описывает команды с фиксированным идентификатором, а шаблоны это список переменной длины. Подменю строится из `DocumentTemplates.available`, а не из `CommandID`.

### Выделение имени без расширения — уже готово

При старте инлайн-редактора должно выделяться имя без расширения: иначе первый же набранный символ стирает `.docx`, и файл перестаёт открываться.

**Работы здесь нет.** `FileListView.Coordinator.selectNameWithoutExtension` уже делает это (`Sources/Pathway/FileListView.swift:347`) и вызывается из `beginRenamingIfNeeded`; для папок выделение остаётся полным. Раздел оставлен, чтобы при реализации это не переделывали заново — достаточно проверить пунктом ручного чек-листа.

## Поток данных

1. **Старт приложения.** `PathwayApp` создаёт `InstalledApps()`; тот опрашивает Launch Services по шести bundle ID и запоминает найденные. `AppState` держит его как `appLookup`.
2. **Правый клик.** `menuNeedsUpdate` зовёт `DocumentTemplates.available(with:)` — чтение готового `Set`, обращений к системе нет.
3. **Выбор пункта.** `browser.createDocument(template)` копирует заготовку через `FileOperations`.
4. **Обратный канал.** Модель отдаёт URL нового файла, `AppState.pendingRename` получает его — существующий механизм, тот же, что после «Новой папки».
5. **Список.** `run { }` инвалидирует кэш каталога и перезагружает — как у всех файловых операций.
6. **Редактор.** `FileListView` открывает инлайн-редактор на новой строке и выделяет имя без расширения.

## Ручной чек-лист

1. Правый клик по пустому месту → «Создать» → «Текстовый документ»: файл появился, имя в режиме редактирования, расширение `.txt` **не** выделено.
2. Набрать имя, Enter → файл переименован, расширение на месте, открывается в TextEdit.
3. Создать «Документ Word» → файл открывается в Word без жалоб на повреждение. То же для Excel и PowerPoint.
4. Создать «Документ Pages» → открывается в Pages. То же для Numbers и Keynote.
5. Создать два документа подряд, не переименовывая: второй называется «Новый документ 2».
6. Escape в редакторе → файл остался с именем по умолчанию, не удалён.
7. На машине без Office блока Microsoft в меню нет, лишнего разделителя тоже.
8. Смонтировать том только для чтения → пункты подменю неактивны.
9. `⇧⌘N` по-прежнему создаёт папку.
10. Правый клик по файлу (не по пустому месту) → подменю «Создать» на месте и работает в текущей папке.

## Вне рамок

- **Пункт в главном меню «Файл».** Контекстное меню — виндовый сценарий, ради которого задача и ставилась. Дублирование в строке меню можно добавить позже, оно ничего не ломает.
- **Пользовательские шаблоны** (папка `~/Templates`, как `Shell New` в Windows). Заметная работа с настройками и UI; спрошено не было.
- **Шорткаты для документов.** Свободных сочетаний нет, а восемь пунктов их и не требуют.
- **Создание из копии выделенного файла** («Создать → как этот»). Другая механика, отдельная задача.
- **Отслеживание установки Office на лету.** Осознанно: см. `InstalledApps`.
- **Заготовки для форматов старше Office 2007** (`.doc`, `.xls`). Word открывает `.docx` двадцать лет; плодить пункты незачем.
